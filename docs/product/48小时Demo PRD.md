## 一、Demo基础信息

| 项目 | 内容 |
|------|------|
| **产品名称** | 《异次元梦想局》- Demo版 |
| **开发周期** | 48小时 |
| **目标** | 验证核心玩法和玩家体验，获取用户反馈 |
| **交付物** | 可运行的Web Demo（PC+移动端） |


## 二、Demo范围界定

### 2.1 核心验证目标

| 编号 | 验证目标 | 验证方式 | 通过标准 |
|------|---------|---------|---------|
| V1 | 玩家是否对“预设角色”产生代入感 | 测试后访谈 | ≥4/5玩家表示“能理解角色为何会动心” |
| V2 | 玩家是否在“崩塌”时刻产生情感冲击 | 观察+问卷 | ≥4/5玩家在崩塌时感到“被触动” |
| V3 | 核心对话机制是否流畅 | 操作计时 | 平均单次对话选择≤10秒 |
| V4 | 玩家是否理解“不收集真实信息” | 测试后访谈 | ≥4/5玩家意识到“没有输入真实信息” |

### 2.2 In Scope（48小时内完成）

| 模块 | 内容 | 工作量 |
|------|------|--------|
| 角色选择 | 3个预设角色（林晓/张伟/王磊） | 4h |
| 第一幕「希望」 | 仿微信聊天界面 + 预设话术树 | 8h |
| 第二幕「崩塌」 | 简单后果展示 + 1个真实案例弹窗 | 4h |
| 第三幕「废墟」 | 2个行动按钮（报警/心理支持） | 4h |
| 第四幕「回放」 | 简单时间线回放（3个节点） | 4h |
| 第五幕「盾牌」 | 简单报告 + 防骗工具箱 | 4h |
| 基础UI | 移动端优先的响应式设计 | 8h |
| 部署 | CloudBase静态托管 | 2h |
| 缓冲 | 预留时间 | 10h |

### 2.3 Out of Scope（Demo不做）

- AI实时生成对话（使用预设话术树代替）
- 全部11条线（仅3条：杀猪盘/刷单返利/求职诈骗）
- 全部11个角色（仅3个）
- 蝴蝶效应机制、外部干预系统、求助网络系统
- 真实用户数据存储（仅LocalStorage）


## 三、Demo功能详细设计

### 3.1 角色选择（3个角色）

| 角色 | 姓名 | 年龄 | 职业 | 匹配线 |
|------|------|------|------|--------|
| R1 | 林晓 | 28 | 互联网运营 | 杀猪盘 |
| R2 | 张伟 | 35 | 制造业主管 | 虚假投资理财 |
| R3 | 王磊 | 24 | 应届毕业生 | 刷单返利 |

**交互**：3张角色卡片 → 点击展开档案 → 确认选择

### 3.2 第一幕：对话系统

**交互方式**：
- 快捷选项为主（3-4个预设回复按钮）
- 辅以自由文本输入（但系统会过滤替换真实信息）

**话术树结构**（以杀猪盘-林晓为例）：

```
Day 1（3轮对话）：
  骗子：“你好呀，看你的头像好亲切，可以认识一下吗？”
  玩家选项：①“好呀” ②“你是？” ③“不了谢谢”
  → 选择①②继续，选择③进入警觉结局

Day 2（4轮对话）：
  骗子：“我平时工作太忙了，很少主动加人。”
  骗子：“我一个人在外地打拼，有时候挺孤独的。”
  玩家选项：分享角色信息/敷衍/拒绝

Day 3（4轮对话）：
  骗子：“我前女友因为我太忙离开了我……”
  骗子：“（不经意）我舅舅是证券高管……”
  玩家选项：追问/不追问/质疑

Day 4（最终决策）：
  骗子：“你能不能帮我登一下我的账户？”
  玩家选项：①答应操作 ②拉黑 ③假装答应
```
