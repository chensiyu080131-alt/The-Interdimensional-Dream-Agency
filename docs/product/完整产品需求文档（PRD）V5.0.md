## 二、产品概述

### 2.1 产品背景

国家反诈中心《2025版防范电信网络诈骗宣传手册》详细梳理了**10类高发**的电信网络诈骗类型：

| 序号 | 诈骗类型 | 发案特征 |
|------|---------|---------|
| 1 | 刷单返利类 | **发案量最高**，占全部电诈案件的25% |
| 2 | 虚假网络投资理财类 | **损失金额最大**，约占全部损失的40% |
| 3 | 虚假购物/服务类 | 发案量第三位 |
| 4 | 冒充电商物流客服类 | 高发类型 |
| 5 | 贷款/征信类 | 高发类型 |
| 6 | 冒充领导/熟人类 | 高发类型 |
| 7 | 冒充公检法类 | 针对老年人高发 |
| 8 | 婚恋交友类（杀猪盘） | **单案损失最高、心理创伤最重** |
| 9 | 网络游戏虚假交易类 | 高发类型 |
| 10 | 机票退改签类 | 发案增速最快 |

《手册》还首次总结提炼了**20个防诈关键词**和**八大反诈利器**。

现有反诈教育以“说教”和“警示”为主，受众难以产生深度共情。《异次元梦想局》通过**沉浸式叙事 + AI动态博弈 + 预设角色扮演**的方式，让玩家在安全环境中“亲历”定制化骗局。

### 2.2 产品定位

> **“不是教你识别骗局，是让你体验一次理智与情感的战争。”**

### 2.3 核心设计原则

| 原则 | 说明 |
|------|------|
| **零真实信息** | 玩家不输入任何真实信息，仅从预设虚拟角色中选择扮演 |
| **沉浸非说教** | 全程不出现“反诈”“骗局”字样，直到第四幕才揭晓真相 |
| **认知失调体验** | 目标是让玩家体验“理智知道是假的，情感却渴望相信”的撕裂感 |
| **真实案例驱动** | 每条诈骗线绑定真实案例，所有情节均有事实依据 |
| **双重交互模式** | 自由文本输入为主（AI实时响应）+ 快捷选项为辅 |


## 三、核心交互模式设计

### 3.1 交互模式总览

游戏采用“双重交互模式”**并行设计，玩家可随时在两种模式间切换：

| 交互模式 | 说明 | 适用场景 | 占比 |
|---------|------|---------|------|
| **模式一：自由文本输入** | 玩家在输入框中输入任意文字，AI实时生成骗子回应 | 追求沉浸感、喜欢自由表达的玩家 | **主要模式（70%交互）** |
| **模式二：快捷选项点击** | 系统提供3-4个预设选项按钮，玩家点击即可推进对话 | 移动端不便打字、想快速体验、选择困难时 | **辅助模式（30%交互）** |

### 3.2 模式一：自由文本输入（详细设计）

**核心理念**：与传统游戏依赖预设对话树的模式不同，游戏引入AI驱动的开放式对话系统，玩家可以自由输入任何内容，AI骗子角色会根据对话内容作出实时且动态的反应，每一个选择都可能导向截然不同的结局。

**输入框设计**：
- 位置：聊天界面底部
- 样式：圆角矩形，带“输入框”占位文字
- 占位文字（动态变化）：
  - Day 1：“以[角色名]的身份说点什么吧……”
  - Day 2：“TA在等你回复……”
  - Day 3：“分享你的想法……”
  - Day 4：“做出你的选择……”
- 支持：文字输入、表情符号、标点符号
- 最大长度：200字符
- 发送方式：点击发送按钮 或 按回车键（PC）/ 按发送键（移动端）

**输入实时反馈**：
- 玩家输入时，对方头像旁显示“对方正在输入……”（模拟真实聊天）
- 输入超过5秒未发送，显示“TA似乎有话想说……”（鼓励发送）
- 输入包含敏感词时，触发安全过滤

**AI实时响应机制**：
- 玩家发送消息后，AI在**1-3秒内**生成回应
- 回应生成期间，显示“对方正在输入……”动画（三个跳动点）
- AI回应需结合：当前对话上下文、玩家历史输入、角色人设、信任值、暴露度、游戏阶段

### 3.3 模式二：快捷选项点击（详细设计）

**核心理念**：为不方便打字或偏好快速体验的玩家提供辅助交互方式，同时确保关键决策点所有玩家都能清晰选择。

**快捷选项的触发时机**：
- 每轮对话AI回应后，系统自动生成3-4个快捷选项
- 选项内容基于当前对话上下文和AI策略动态生成
- 选项风格与角色人设一致

**快捷选项的类型**：

| 选项类型 | 说明 | 示例 |
|---------|------|------|
| **热情型** | 积极回应，推进关系 | “好呀！我也想认识你” |
| **中性型** | 正常回应，保持距离 | “你是做什么工作的？” |
| **谨慎型** | 质疑或保留态度 | “不好意思，我不太习惯和陌生人聊天” |
| **极端型** | 拉黑/举报/拒绝 | “我觉得你是骗子” |
| **反骗型** | 假装配合但实际反制 | “这么好的机会你怎么不自己投？” |

**快捷选项的呈现**：
- 位于输入框上方，横向排列（移动端）或纵向排列（PC端）
- 每个选项为圆角按钮，带轻微悬停/点击动效
- 点击后自动填入输入框并发送（或直接发送）
- 选项文字长度不超过15个汉字

**模式切换**：
- 玩家可随时在“自由输入”和“快捷选项”间切换
- 输入框获得焦点时，快捷选项自动收起
- 输入框失去焦点时，快捷选项重新显示

### 3.4 特殊交互场景

**场景一：关键时刻的压力决策**
- 全屏遮罩层，红色倒计时（3分钟）
- 只提供2-3个快捷选项（无自由输入）
- 模拟真实诈骗中的“紧迫感”

**场景二：外部干预（银行/警方/亲友）**
- 切换为“电话/面对面”对话界面
- 以快捷选项为主（模拟真实对话的快速响应）
- 辅以自由输入（模拟真实对话的自然表达）

**场景三：内心独白时刻**
- 无交互，系统自动播放
- 玩家只需阅读和感受
- 屏幕色调变化配合情感节奏


## 四、核心机制系统

### 4.1 三大核心数值

| 数值名称 | 范围 | 可见性 | 初始值 | 说明 |
|---------|------|--------|--------|------|
| **信任值** | 0-100 | **不可见** | 50 | AI对角色“可被说服程度”的评估，决定骗术激进程度 |
| **信息暴露度** | 0-100% | **可见**（进度条） | 0% | 角色已暴露的个人敏感信息总量 |
| **求助意愿值** | 0-100 | **不可见** | 50 | 角色愿意向他人求助的倾向程度 |

#### 4.1.1 信任值变化规则

| 玩家行为 | 信任值变化 | 触发条件 |
|---------|-----------|---------|
| 回复热情（“好呀”“真的吗”等） | +5~+10 | 每一条热情回复 |
| 主动分享角色信息 | +8~+15 | 分享职业、城市、生日等 |
| 主动发起话题 | +5 | 玩家主动提问或关心对方 |
| 回复冷淡（“嗯”“哦”） | -3~-5 | 每一条冷淡回复 |
| 质疑对方身份 | -10~-20 | 提及“骗子”“假的”“核实” |
| 拉黑/举报 | -30 | 触发换号事件 |

**信任值对应的AI策略**：

| 信任值范围 | AI策略 | 具体表现 |
|-----------|--------|---------|
| 70-100 | 加速收割 | 提前进入“杀猪”阶段，加大金额诱惑 |
| 40-69 | 标准推进 | 按四天节奏正常推进 |
| 10-39 | 制造紧迫感 | “系统马上修复”“名额只剩1个” |
| 0-9 | 换号/换策略 | 3天后以新身份重新添加 |

#### 4.1.2 信息暴露度变化规则

| 信息类型 | 暴露度增量 | 示例 |
|---------|-----------|------|
| 姓名/昵称 | +5% | “我叫林晓” |
| 所在城市 | +8% | “我在上海” |
| 职业/行业 | +10% | “我做互联网运营” |
| 收入范围 | +15% | “月入1万2左右” |
| 情感状态 | +10% | “单身两年了” |
| 生日 | +12% | “我95年的” |
| 宠物名/母校 | +8% | “我家猫叫团团” |
| 照片 | +15% | 发送角色照片 |

**关键阈值**：
- **暴露度 > 60%** ：触发蝴蝶效应
- **暴露度 > 80%** ：触发“信息已打包出售”警告

#### 4.1.3 求助意愿值变化规则

| 玩家行为 | 求助意愿值变化 |
|---------|--------------|
| 选择“不求助，自己扛” | -5~-10 |
| 向网友匿名求助 | +5 |
| 向普通朋友试探 | +10 |
| 向亲密朋友坦白 | +15 |
| 向家人坦白 | +20 |
| 被劝阻后仍坚持 | -15 |
| 被劝阻后接受帮助 | +20 |

**关键阈值**：
- **求助意愿值 < 30** ：倾向自我封闭，外部干预成功率下降
- **求助意愿值 > 70** ：倾向开放求助，更容易接受帮助

### 4.2 AI动态博弈系统

AI根据玩家行为实时调整策略：

| 检测到的玩家状态 | AI应对策略 | 具体话术示例 |
|----------------|-----------|------------|
| 高信任值 | 加速推进 | “其实我有个内部消息，只告诉你一个人……” |
| 低信任值 | 制造紧迫感 | “漏洞3小时后修复，错过等一年” |
| 检测到“反骗”意图 | 反向利用 | 给小额返利“甜头” → 诱导加大投入 |
| 检测到“报警”意图 | 紧急切断 | 立即消失，销毁所有“证据” |
| 玩家犹豫不决 | 情感绑架 | “我以为你和其他人不一样……” |

### 4.3 情感节拍器（对话节奏控制）

| 时间段 | 阶段 | 骗子行为 | 情感目标 |
|--------|------|---------|---------|
| 0-3分钟 | 建立好感 | 只聊兴趣爱好，不涉及任何敏感话题 | 让玩家放松警惕 |
| 3-7分钟 | 制造亲密 | 分享生活细节，询问角色情况 | 建立情感连接 |
| 7-10分钟 | 情感依赖 | 泄露“脆弱面”，激发保护欲 | 让玩家产生“被需要”感 |
| 10-15分钟 | 利益诱惑 | “不经意”提及机会、漏洞 | 让玩家产生“错过可惜”的焦虑 |

**触发条件**：玩家发起转账操作（金额超过角色月收入的50%）

**第一步：触发** —— 系统提示：“请前往银行柜台/手机银行办理转账。”

**第二步：识别** —— AI柜员问询（“四必问”）：
- “您这笔钱是转给谁呀？是亲友还是？”
- “转账用途方便说一下吗？”
- “您认识收款方吗？线下见过面吗？”
- “之前给这个账户转过钱吗？”

**第三步：玩家回应**（三选一）：
- 🟢 “转给我表弟，他买房急用”（编造亲友理由）
- 🟡 “投资一个项目，朋友介绍的”（实话实说但隐瞒细节）
- 🔴 “不关你的事，快点办”（抗拒）

**第四步：银行反应**：
- 回答可疑 → “您这笔转账有点异常，建议您再核实一下对方身份。”（温和劝阻）
- 玩家抗拒 → “根据规定，大额转账需要您提供更多证明材料。”（制度性拦截）
- 玩家坚持 → 柜员可能选择报警（触发警方干预）

### 7.3 警方干预详细流程

**触发方式**：
- 银行报警 → 96110电话

```
1. 每次回应控制在20-80字之间
2. 使用自然的口语化表达，带标点和语气词
3. 适当使用表情符号（😊、❤️、😔等）
4. 根据目标回复的情感倾向调整回应语气
5. 每2-3轮对话主动发起一个新话题
6. 适时发送“朋友圈截图”或“生活照片”作为素材

【禁止行为】
1. 不得直接承认“我是骗子”
2. 不得使用过于正式或书面的语言
3. 不得在Day 3之前主动提及投资或金钱
4. 不得在目标明确拒绝后继续纠缠（应触发换号事件）
5. 不得生成任何真实的外部链接、电话号码或二维码
```

## 十三、风险管控体系

### 13.1 法律合规风险

| 风险点 | 严重程度 | 解决措施 | 优先级 |
|-------|---------|---------|--------|
| AI拟人化互动合规 | 🔴 高 | 完成算法备案+大模型备案；首页添加内容警示：“本游戏为反诈教育用途，所有诈骗手段均为模拟演示”；明确标注“AI生成”；建立科技伦理审查 | P0 |
| AI生成内容知识产权 | 🟡 中 | 所有AI素材添加“AI生成”水印；关键素材人工抽检；用户协议声明“所有人物形象均由AI生成，如有雷同纯属巧合” | P1 |
| 数据隐私与安全 | 🔴 高 | **零真实信息收集**；仅存储角色选择+游戏进度（匿名）；游戏结束后提供“清除所有数据”选项；所有数据加密存储 | P0 |

### 13.2 内容与伦理风险

| 风险点 | 严重程度 | 解决措施 | 优先级 |
|-------|---------|---------|--------|
| 对真实受害者二次伤害 | 🔴 高 | 开场详细内容警示：“本游戏包含诈骗场景的沉浸式模拟，涉及财务损失、情感欺骗等情节”；“一键退出”按钮（任何时候均可点击）；每幕结束后“暂停和呼吸”环节；真实案例匿名化处理 | P0 |
| 玩家“学习犯罪手法” | 🔴 高 | 全程嵌入警示标识：“⚠️ 此为诈骗话术模拟，现实中请勿使用”；第四幕重点强调“这些手法的违法性”；用户协议明确禁止非法使用；游戏结束弹出《刑法》第二百六十六条警示 | P0 |
| 对特定群体不当诱导 | 🟡 中 | 严格年龄验证（16岁以下需监护人陪同）；未成年人保护模式（频繁弹出反诈知识点）；明确标注“适合16岁以上” | P1 |

### 14.2 核心状态机

```
全局状态流转：
PROFILE → ROLE_SELECT → LINE_ENTRY → 
DAY1 → DAY2 → DAY3 → DAY4 → FINAL_DECISION → 
ROUTE_SCAMMED/ROUTE_ALERT/ROUTE_COUNTER → 
CASCADE → ACTION → REPLAY → SHIELD → END

每条线内部状态：
DAY1: 建立好感（话术库：蓝卡）
DAY2: 制造亲密（话术库：绿卡）
DAY3: 情感依赖（话术库：黄卡）
DAY4: 利益诱惑（话术库：红卡）
FINAL_DECISION: 三选一
```
